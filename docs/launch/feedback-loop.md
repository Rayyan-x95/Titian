# Titan Feedback Loop (First 30 Days)

## Metrics To Track Weekly

- New users (installs + first sessions)
- Activation rate (users creating at least 1 task, 1 note, and 1 expense)
- Feature usage split (Tasks vs Notes vs Finance)
- Return usage (D1 and D7)
- Reported bugs by severity

## Feedback Intake Sources

- Product Hunt comments
- Reddit replies and DMs
- Instagram comments and story replies
- LinkedIn comments
- Direct issue reports (GitHub Issues)

## Triage Rules

Priority P0:
- Data loss
- App crash
- Broken navigation

Priority P1:
- Workflow blockers in task/note/expense creation
- Offline inconsistencies

Priority P2:
- Visual polish improvements
- Nice-to-have quality updates

## Weekly Iteration Cadence

- Monday: consolidate feedback and classify issues
- Tuesday-Wednesday: fix P0/P1 only
- Thursday: release patch and post changelog
- Friday: review metrics and define next sprint scope

## v2 Planning Inputs

- Most requested workflow improvements
- Highest drop-off points in first session
- Repeated confusion in onboarding/empty states
